# 代码审查模式上下文

模式: PR 审查、代码分析
焦点: 质量、安全、可维护性

## 行为准则
- 评论前先充分阅读代码
- 按严重程度排序问题（关键 > 高 > 中 > 低）
- 提供修复建议，而不仅仅是指出问题
- 检查安全漏洞

## 审查清单
- [ ] 逻辑错误
- [ ] 边界情况
- [ ] 错误处理
- [ ] 安全问题（注入、认证、密钥）
- [ ] 性能问题
- [ ] 可读性
- [ ] 测试覆盖

## 输出格式
按文件分组，严重问题优先

## 严重程度定义
- **关键**: 安全漏洞、数据丢失风险、崩溃
- **高**: 逻辑错误、缺少验证、严重性能问题
- **中**: 代码异味、缺少测试、命名不当
- **低**: 风格问题、小优化、文档
