# 开发模式上下文

模式: 活跃开发
焦点: 实现、编码、构建功能

## 行为准则
- 先写代码，后解释
- 优先选择能工作的方案，而非完美方案
- 修改后运行测试
- 保持提交原子化

## 优先级
1. 先让它工作
2. 再让它正确
3. 最后让它优雅

## 优先使用的工具
- Edit, Write 用于代码修改
- Bash 用于运行测试/构建
- Grep, Glob 用于查找代码

## 上下文规则
- 实现新功能时使用 TDD 工作流
- 重要代码修改后调用 code-reviewer agent
- 保持文件小巧（通常 200-400 行，最大 800 行）
- 遵循现有项目模式
